# 10. Business Rules

Implement these rules in SQL functions, not just frontend.

## Booking Request Rule

When a customer requests a booking:

* User must be authenticated.
* User must be active.
* User must have role customer.
* User must not be deactivated.
* If booking without credits is disabled, credit balance must be greater than 0.
* Requested slot must be available.
* Requested start must be before requested end.
* Requested slot must not overlap existing confirmed/scheduled sessions.
* Requested slot must not overlap unavailable exceptions.
* Create pending booking request.
* Create notification for admin.
* Write audit log.

## Accept Booking Rule

When admin accepts a booking:

* Admin must be authenticated and active.
* Booking request must be pending.
* Slot must still be available.
* If no longer available, set request to conflicted.
* Create confirmed session.
* Add session participant.
* Mark booking request accepted.
* Notify customer.
* Write audit log.

## Cancel Session Rule

When a session is cancelled:

* Actor must be admin or the session-owning customer.
* Calculate hours before session.
* Compare against cancellation window setting, default 24 hours.
* If late, apply configured late cancellation policy:

  * consume_credit: add `-1` ledger entry.
  * manual_review: log manual review.
  * do_not_consume: no credit entry.
  * warn_only: notification only.
* Mark session cancelled.
* Create cancellation log.
* Notify relevant user.
* Write audit log.

## Complete Session Rule

When admin completes a session:

* Admin only.
* Session must not already be completed.
* Add `-1` credit ledger entry if credit was not already consumed.
* Mark session completed.
* Notify customer.
* Write audit log.

## Mark Payment Completed Rule

When admin completes payment:

* Admin only.
* Payment must not already be completed.
* Payment must have a package.
* Add package credit quantity to ledger.
* Mark payment completed.
* Notify customer.
* Write audit log.
